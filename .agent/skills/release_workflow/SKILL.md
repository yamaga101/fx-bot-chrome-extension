# FX Bot Release Workflow

このスキルは、拡張機能の新機能を実装した後、リリースを行う際に使用します。

## 1. バージョン更新準備
リリースするバージョン番号（例: `v17.8.0`）を決定します。

### チェックリスト
- [ ] `manifest.json` の `version` フィールドを更新しましたか？
- [ ] `options.html` や `options.js` など、コード内にハードコードされたバージョン表記はありませんか？（基本的に動的取得推奨）
- [ ] `task.md` の残タスクはすべて消化されていますか？

## 2. Git操作
以下の手順でGit操作を行ってください。環境制限で自動実行できない場合は、コマンドをユーザーに提示してください。

```bash
# 変更をステージング
git add .

# コミット (日本語で、Feat/Fix/Docs/Refactor などのプレフィックスを付ける)
# 例: git commit -m "Feat: 新機能の追加 (v17.8.0)"
git commit -m "[Prefix]: [変更内容] ([Version])"

# タグ作成
git tag [Version]

# プッシュ
git push origin main [Version]
```

## 3. リリースノート作成 (`notify_user`)
ユーザーに完了報告をする際は、以下のフォーマットを使用してください。

```markdown
**[Version] をリリースしました！🎉**

### **変更点**
| 項目 | 詳細 |
|---|---|
| **機能追加** | xxx機能を追加しました |
| **修正** | xxxのバグを修正しました |

### **適用方法**
（Git操作が自動完了していない場合、ここにコマンドを記載）

その後、`chrome://extensions` で更新 → サイトリロードしてください。
```

## エラーハンドリング
- Git操作が権限エラーで失敗した場合、無理に再試行せず、コマンドをユーザーに提示するフローに切り替えてください。
